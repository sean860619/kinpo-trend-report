# 早晨日報 /morning

執行以下步驟，彙整成一份早晨日報給使用者。

## 資料收集步驟

### 1. Gmail — 昨日重要郵件
使用 Gmail MCP 搜尋昨天收到的郵件：
- 搜尋條件：`after:昨天日期 before:今天日期`（格式 yyyy/mm/dd）
- 重點篩選：有回覆需求、重要通知、待處理事項
- 忽略：行銷廣告、自動通知信（newsletters、noreply）

### 2. Google Calendar — 今日行程
使用 Google Calendar MCP 讀取今天的所有行程：
- 列出所有今日活動（含時間、地點、參與者）
- 標示需要提前準備的會議

### 3. Notion — 待辦與任務
使用 Notion MCP 搜尋：
- 未完成的任務（status 非 Done/完成）
- 今天到期或過期的任務
- 最近更新的頁面（昨天有動過的）

## 輸出格式

用以下格式輸出，語言使用繁體中文：

---

# ☀️ 早晨日報｜$TODAY

## 📬 昨日郵件摘要
（列出重要郵件，每封一行，格式：寄件者 — 主旨 — 行動建議）
若無重要郵件，寫「無需處理的重要郵件」

## 📅 今日行程
（依時間排序，格式：HH:MM 事件名稱 [地點/連結]）
若無行程，寫「今日無行程」

## ✅ 待辦重點
（列出今日最重要的 3–5 件待辦，來自 Notion）
若無，寫「Notion 無待辦任務」

## 💡 今日建議
（根據以上資訊，給出 1–2 句簡短的工作建議或提醒）

---

注意：
- 日期請自動計算（今天 = 執行當下的日期，昨天 = 前一天）
- 若某個服務讀取失敗，在該區塊標示「⚠️ 無法讀取，請確認連線」
- 整份日報力求簡潔，讓人 2 分鐘內看完
