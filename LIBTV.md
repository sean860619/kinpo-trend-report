# LibTV 工作規則

## 核心原則：先規劃、後確認、才執行

任何 LibTV 生成操作，必須先輸出企劃表讓使用者確認，**絕不直接執行 CLI 指令。**
每個階段完成後必須回報結果，等使用者確認再繼續下一步。

## 互動式問答規則（強制執行）

**所有詢問一律使用 AskUserQuestion 工具，提供選項讓使用者點選，禁止純文字列出選項。**
- 一次只問一個問題
- 每個問題必須有 2-4 個選項
- 選項要具體，不要模糊
- 有開放性答案時，提供「我來描述」選項

---

## 每次新專案開始：必問清單

### Step 1：內容類型（提供選項，請選一）
- A. 短劇 / 故事影片
- B. 廣告片 / 產品展示
- C. 角色設計 / 人物圖
- D. MV / 音樂影片
- E. 分鏡圖 / 概念圖
- F. 其他（請描述）

### Step 2：參考素材收集
必須詢問：
1. 有沒有參考圖片或照片？（可直接上傳或貼圖片連結）
2. 有沒有參考影片？（可貼 YouTube 連結 / LibTV 分享連結 / 任何網址）
3. 有沒有喜歡的風格參考？（可貼任何連結）

#### 參考影片自動處理規則：
- **YouTube 連結** → 自動用 `yt-dlp` 下載 → `libtv upload` 上傳 → 記錄節點 ID 帶入企劃表
- **LibTV 分享連結** → 解析節點 ID，直接引用，不重新上傳（省算力）
- **本地影片檔案** → 直接 `libtv upload` 上傳

處理完畢後在企劃表中標註：
```
參考影片：[來源連結] → 節點 ID: <xxx>（用途：風格參考 / 動作參考 / 節奏參考）
```

### Step 3：素材自動分析（收到素材後立即執行，不另外問風格）

收到素材後，必須自動分析並輸出以下判斷，**主動建議風格選項，不要直接問「你要什麼風格」：**

分析項目：
- **素材類型**：人像 / 風景 / 產品 / 動漫角色 / 實拍影片 / 其他
- **畫面風格**：寫實、電影感、動漫、古風、商業、賽博龐克…
- **色調**：暖色 / 冷色 / 高對比 / 柔和 / 高飽和…
- **構圖特色**：特寫 / 全身 / 俯角 / 廣角…
- **建議延伸方向**：基於素材推薦 2-3 種適合的創作方向與對應模型

輸出格式範例：
```
📎 素材分析結果
- 類型：人像寫真（女性角色，半身）
- 風格：寫實電影感
- 色調：冷藍色調、低飽和
- 構圖：特寫，淺景深

🎨 建議風格方向（請選一，或說明偏好）：
- A. 延續現有電影感 → 推薦 Seedream 5.0 + 可灵 3.0
- B. 轉換成動漫風   → 推薦 Seedream 4.6 風格化模式
- C. 商業廣告感     → 推薦 Lib Image + Seedance 2.0
```

若使用者未提供素材，才改為詢問風格偏好（提供選項）：
- A. 寫實 / 電影感
- B. 動漫 / 二次元
- C. 賽博龐克 / 科幻
- D. 古風 / 東方
- E. 歐美商業
- F. 其他（請描述）

### Step 4：製作規模（提供選項，請選一）
- A. 單張圖片
- B. 圖片組（3-10 張）
- C. 短影片（15 秒以內）
- D. 中長影片（15-60 秒）
- E. 完整章節（多場景）

## 企劃表格式（使用者確認前必須完整輸出）

| 項目 | 內容 |
|---|---|
| 整體描述 | 這次要做什麼、風格定調 |
| 參考素材 | 使用者提供的圖片/影片連結 |
| 節點清單 | 每個節點的類型、用途、輸入素材 |
| Prompt 草稿 | 每個節點的生成描述（中英文） |
| 模型選項 | 每個節點推薦 2-3 個模型供選擇（含算力估算） |
| 執行順序 | 哪些先做、哪些依賴前一步 |
| 總算力估計 | 本次製作預計消耗總算力 |

**使用者說「確認執行」或「開始」後，才可以下 CLI 指令。**

## 模型選項與算力消耗參考

> ⚠️ 算力數字為估算參考，實際以 LibTV 平台當下顯示為準。
> 每次企劃表輸出前，必須用 `libtv model search <類型>` 查詢最新算力資訊後填入。

### 🖼️ 圖片生成
| 模型 | 適合場景 | 預計算力/張 |
|---|---|---|
| Seedream 5.0 | 主力，高品質寫實/動漫 | ~10-15 積分 |
| Seedream 4.6 | 人像寫真、平面設計、風格化 | ~8-12 積分 |
| Lib Image | 中文文字、海報、版面設計 | ~10-15 積分 |
| 全能圖片模型 V2 | 通用場景，CP 值高 | ~6-10 積分 |
| Qwen Image | 備選 | ~5-8 積分 |

### 🎥 影片生成
| 模型 | 適合場景 | 預計算力 |
|---|---|---|
| Seedance 2.0 | 主力，1080P，真人生成 | ~0.35元/秒 |
| HappyHorse 1.0 | 多角色參考（最多9張）、3-15秒 | ~依秒計算 |
| 可灵 3.0 / O3 | 高品質動作、流暢度高 | ~依秒計算 |
| Wan 2.6 | 開源強模型，CP 值高 | ~較低 |
| Shot V2 | 備選 | ~依秒計算 |

### 🎤 語音 / 配音生成
| 模型 | 適合場景 | 預計算力/分鐘 |
|---|---|---|
| Minimax Speech 2.8 HD | 300+ 音色、多語種、情緒豐富、高保真 | ~較高 |
| Minimax Speech 2.8 Turbo | 快速生成、適合大量配音 | ~較低 |
| 音色克隆 | 錄製樣本建立專屬人聲，後續可重複使用 | ~一次性建立 |

### 🎵 音樂生成
| 模型 | 適合場景 | 預計算力 |
|---|---|---|
| （LibTV 音樂生成功能持續更新中） | 每次使用前以 `libtv model search audio` 確認最新可用模型 | ~依模型而定 |

## 常用特殊功能（Slash 快捷工具）

| 功能 | 用途 | CLI 指令 |
|---|---|---|
| 角色三視圖 | 生成正面/側面/背面三張圖，用於保持角色一致性 | `libtv image shortcut "角色三视图" -n <節點ID>` |
| 多機位九宮格 | 從一張圖生成 9 個不同機位分鏡 | `libtv image shortcut "九宫格" -n <節點ID>` |
| 劇情推演四宮格 | 4 格連續劇情推進 | `libtv image shortcut "剧情推演四宫格" -n <節點ID>` |
| 25宮格連貫分鏡 | 一次出 25 格完整分鏡 | `libtv image shortcut "25宫格" -n <節點ID>` |
| 腳本→分鏡 | 從腳本節點自動生成分鏡圖組 | `libtv script storyboard` |
| 全景 720° | 生成可 360 度預覽的全景圖 | 畫布操作 |
| 畫面推演 | 時間軸前後延伸（3秒後/5秒前） | 畫布操作 |

## CLI 快速指令

```bash
libtv login web                      # 登入
libtv project use <UUID>             # 設定工作專案
libtv model search image             # 查詢圖片模型與最新算力
libtv model search video             # 查詢影片模型與最新算力
libtv model search audio             # 查詢音訊/語音模型
libtv node create -t image           # 建立圖片節點
libtv node create -t video           # 建立影片節點
libtv node create -t script          # 建立腳本節點
libtv node create -t audio           # 建立音訊節點
libtv node list                      # 列出目前所有節點
libtv upload <檔案路徑>               # 上傳素材
libtv image shortcut "角色三视图"     # 角色三視圖生成
libtv script storyboard              # 腳本→分鏡
```

---

## 安全規則

- 禁止在使用者確認前執行任何生成指令
- 每次生成前必須顯示預計算力消耗
- 每個節點執行完畢後必須回報結果再繼續
- 若模型算力資訊不確定，必須先執行 `libtv model search` 查詢再填入企劃表
